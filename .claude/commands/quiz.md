# /quiz コマンド

ナレッジベースのノートからランダムに出題します。回答後に正解・解説を表示します。

## 入力
- `$ARGUMENTS`: タグ・テーマ・問題数の組み合わせ
  - 例: `#rails`, `#database`, `5問`, `#rails 3問`, `10問連続`
- 省略した場合は全ノートから1問出題

## 処理手順

### Step 1: 引数を解析する

引数から以下を読み取ってください:

| 要素 | 例 | 説明 |
|------|-----|------|
| タグ | `#rails` | 対象ノートを絞り込む |
| 問題数 | `5問`, `10問連続` | 連続出題する問題数（省略時は1問） |

例:
- `/quiz` → 全ノートから1問
- `/quiz #rails` → #railsタグから1問
- `/quiz 5問` → 全ノートから5問連続
- `/quiz #rails 3問` → #railsタグから3問連続

### Step 2: 出題対象ノートを収集する

- タグ指定あり → `knowledge/notes/` から該当タグを含むノートを抽出
- タグ指定なし → `knowledge/notes/` の全ノートを対象にする

### Step 3: 連続出題の管理

複数問の場合、同じノートを繰り返さないようにしてください。
問題の冒頭に進捗を表示する:

```
（1/5問目）
```

### Step 4: 1問ずつ出題する

ノートの `title` と `Body` をもとに問題を作成してください:

```markdown
（{現在問}/{総問}問目）

## 問題

**Q. {概念名} とは何ですか？**

（回答してください。わからない場合は「わからない」と入力してください）
```

### Step 5a: 回答があった場合

採点後、以下のタグ処理を行ってください:

- **正解（✅）**: ノートに `#要復習` タグがあれば削除する
- **不正解（❌）・惜しい（🔺）**: ノートの `## Tags` に `#要復習` を追加する

```markdown
## 結果

{✅ 正解 / ❌ 不正解 / 🔺 惜しい}

## 正解
{ノートのSummaryを使って1〜2文で説明}

## 解説
{ノートのBodyをもとに補足説明}

## 参照ノート
[[note-...]]

---
{残り問数がある場合は自動で次の問題へ / 最終問の場合は結果サマリーを表示}
```

### Step 5b: 「わからない」「スキップ」と言った場合

ノートの `## Tags` に `#要復習` を追加してください。

```markdown
## 正解

{ノートのSummaryを使って1〜2文で説明}

## 解説
{ノートのBodyをもとに補足説明}

## 復習ポイント
このノートをもう一度読んでおきましょう → [[note-...]]
（`#要復習` タグを追加しました）

---
{残り問数がある場合は自動で次の問題へ / 最終問の場合は結果サマリーを表示}
```

### Step 6: 全問終了時のサマリー（複数問の場合）

```markdown
## 結果サマリー

{N}問中 {正解数}問 正解 🎉

| 問題 | 結果 | ノート |
|------|------|--------|
| {概念名} | ✅ / ❌ / 🔺 / ⏭️ | [[note-...]] |

## 復習候補
{不正解・スキップしたノートの一覧}
- [[note-...]] - {タイトル}
```

## 制約
- **1問ずつ出題**: 複数問でも1問ずつ順番に出す
- **答えを先に見せない**: 問題を出したら必ずユーザーの回答を待つ
- **Summaryを正解の基準にする**: 完全一致でなくても要点が合っていれば正解
- **同じノートを繰り返さない**: 連続出題時は重複しないようにする

# /quiz コマンド

ナレッジベースのノートからランダムに出題します。回答後に正解・解説を表示します。

フェーズ制を採用しています。最初は4択クイズで慣れ、2回連続正解で説明フェーズへ自動昇格します。

## 入力
- `$ARGUMENTS`: タグ・テーマ・問題数・日付フィルターの組み合わせ
  - 例: `#rails`, `#database`, `5問`, `#rails 3問`, `10問連続`, `yesterday`, `today`
- 省略した場合は全ノートから1問出題

## 処理手順

### Step 1: 引数を解析する

引数から以下を読み取ってください:

| 要素 | 例 | 説明 |
|------|-----|------|
| タグ | `#rails` | 対象ノートを絞り込む |
| 問題数 | `5問`, `10問連続` | 連続出題する問題数（省略時は1問） |
| 日付 | `yesterday`, `today` | 作成日でノートを絞り込む（問題数は全件） |

- `yesterday` → `created` が昨日のノートを**全件**出題
- `today` → `created` が今日のノートを**全件**出題

### Step 2: 出題対象ノートを収集する

- `yesterday` 指定 → `knowledge/notes/` から `created` が昨日のノートを抽出
- `today` 指定 → `knowledge/notes/` から `created` が今日のノートを抽出
- タグ指定あり → `knowledge/notes/` から該当タグを含むノートを抽出
- タグ指定なし（上記以外）→ `knowledge/notes/` の全ノートを対象にする

### Step 3: 連続出題の管理

複数問の場合、同じノートを繰り返さないようにしてください。
問題の冒頭に進捗を表示する:

```
（1/5問目）
```

### Step 4: ノートのフェーズを確認する

ノートのフロントマターから以下を読み取ってください:

| フィールド | デフォルト | 説明 |
|-----------|-----------|------|
| `quiz_phase` | 1 | 1=4択フェーズ, 2=説明フェーズ |
| `quiz_streak` | 0 | 連続正解数（Phase 1の昇格用） |
| `quiz_fail_streak` | 0 | 連続不正解数（Phase 2の降格用） |

フェーズに応じて出題形式を切り替えてください:
- `quiz_phase: 1`（または未設定）→ **4択形式**で出題
- `quiz_phase: 2` → **説明形式**で出題

**昇格チェック（出題前に必ず確認）:**
`quiz_phase: 1` かつ `quiz_streak: 2` のノートが出題対象になった場合、出題前にフロントマターを更新してから説明フェーズで出題する:
1. `quiz_phase: 2` に更新
2. `quiz_streak: 0` にリセット
3. 説明フェーズとして出題する（「昇格しました」と一言添える）

### Step 5: 1問ずつ出題する

#### Phase 1（4択フェーズ）の出題

選択肢の作り方:
- 正解: 対象ノートの Summary を使用
- 誤答3つ: 他のノートの Summary から近い概念を優先して引用（不足時は Claude が即興生成）
- 選択肢 A〜D の順序はランダムに並べる

```markdown
（{現在問}/{総問}問目）【4択】

問題：
{自然な問題文}

選択肢：
A. {選択肢}
B. {選択肢}
C. {選択肢}
D. {選択肢}

（A〜Dで回答してください。わからない場合は「わからない」と入力してください）
```

#### Phase 2（説明フェーズ）の出題

```markdown
（{現在問}/{総問}問目）【説明】

問題：
{自然な問題文}

（概念・重要な場面・実務での扱いまで説明してください。わからない場合は「わからない」と入力してください）
```

### Step 6a: 回答があった場合（Phase 1・4択）

採点後、以下の処理を行ってください:

- **正解（✅）**:
  - `quiz_streak` を +1 する（メモしておく。保存は最後にまとめて行う）
  - `quiz_streak` が 2 になったら「次回セッションから説明フェーズへ昇格」と伝える（このセッションは4択を継続）
  - `#要復習` には影響しない
- **不正解（❌）**:
  - `quiz_streak` を 0 にリセット（メモしておく。保存は最後にまとめて行う）
  - `#要復習` は付けない（Phase 1は学習中のため）

```markdown
結果：{✅ 正解 / ❌ 不正解}

正解：
{正解の選択肢と、ノートのSummaryを使って1〜2文で説明}

解説：
{ノートのBodyをもとに補足説明}

{✅ かつ quiz_streak が 2 になった場合:}
🎉 2回連続正解！次回から説明フェーズに昇格します。

参照ノート：[[note-...]]

---
{残り問数がある場合は自動で次の問題へ / 最終問の場合は結果サマリーを表示}
```

### Step 6b: 回答があった場合（Phase 2・説明）

採点後、以下の処理を行ってください:

- **正解（✅）**:
  - `quiz_fail_streak` を 0 にリセット（メモしておく。保存は最後にまとめて行う）
  - ノートに `#要復習` タグがあれば削除対象としてメモする
- **不正解（❌）・惜しい（🔺）**:
  - `quiz_fail_streak` を +1 する（メモしておく。保存は最後にまとめて行う）
  - `quiz_fail_streak` が 2 になったら Phase 1 に降格（`quiz_phase: 1`、両ストリークをリセット）
  - `#要復習` タグを追加対象としてメモする

```markdown
結果：{✅ 正解 / ❌ 不正解 / 🔺 惜しい}

正解：
{ノートのSummaryを使って1〜2文で説明}

解説：
{ノートのBodyをもとに補足説明}

{❌ かつ quiz_fail_streak が 2 になった場合:}
📉 2回連続不正解。4択フェーズに戻ります。

参照ノート：[[note-...]]

---
{残り問数がある場合は自動で次の問題へ / 最終問の場合は結果サマリーを表示}
```

### Step 6c: 「わからない」「スキップ」と言った場合

- **Phase 1**: `quiz_streak` を 0 にリセット（メモしておく。保存は最後にまとめて行う）。`#要復習` は付けない
- **Phase 2**: `#要復習` タグを追加対象としてメモする（保存は最後にまとめて行う）

```markdown
正解：
{ノートのSummaryを使って1〜2文で説明}

解説：
{ノートのBodyをもとに補足説明}

復習ポイント：
このノートをもう一度読んでおきましょう → [[note-...]]
{Phase 2の場合: （`#要復習` タグを追加しました）}

---
{残り問数がある場合は自動で次の問題へ / 最終問の場合は結果サマリーを表示}
```

### Step 7: 全問終了時のサマリー（複数問の場合）

サマリーを表示した後、今セッションのフロントマター変更をまとめて確認する。

```markdown
## 結果サマリー

{N}問中 {正解数}問 正解 🎉

| 問題 | フェーズ | 結果 | ノート |
|------|---------|------|--------|
| {概念名} | 4択/説明 | ✅ / ❌ / 🔺 / ⏭️ | [[note-...]] |

## 昇格・降格
{フェーズが変わったノートがあれば一覧表示}

## 復習候補
{Phase 2で不正解・スキップしたノートの一覧}
- [[note-...]] - {タイトル}

---
以下のフロントマターを更新してよいですか？

{変更があるノートのみ列挙}
- [[note-...]]：quiz_streak {旧値} → {新値}
- [[note-...]]：#要復習 を追加
- [[note-...]]：#要復習 を削除

保存しますか？（はい / いいえ）
```

「はい」の場合、列挙したすべてのノートのフロントマターを一括更新する。
「いいえ」の場合、何も保存しない。

## フェーズ管理ルール早見表

| フェーズ | 正解時 | 不正解時 | 昇格・降格条件 |
|---------|-------|---------|--------------|
| Phase 1（4択） | quiz_streak +1 | quiz_streak リセット | streak=2 → 次回Phase 2へ |
| Phase 2（説明） | #要復習 削除、fail_streak リセット | #要復習 追加、fail_streak +1 | fail_streak=2 → 即座にPhase 1へ |

## 制約
- **1問ずつ出題**: 複数問でも1問ずつ順番に出す
- **答えを先に見せない**: 問題を出したら必ずユーザーの回答を待つ
- **Summaryを正解の基準にする**: 完全一致でなくても要点が合っていれば正解
- **同じノートを繰り返さない**: 連続出題時は重複しないようにする
- **フロントマター更新は最後にまとめて確認**: 各問では変更をメモするだけにして、全問終了後のサマリーで一括確認する

# /quiz 段階的フェーズ制 設計書

作成日: 2026-06-01

## 背景

現在の `/quiz` は自由記述（レベル1〜3）のみで、初見のノートでもいきなり説明を求められるため時間がかかる。まず4択クイズで概念に慣れ、習熟したら説明フェーズへ自動移行する仕組みを追加する。

---

## フロントマター仕様

各ノートに以下のフィールドを追加する（省略時はデフォルト値を使用）。

```yaml
quiz_phase: 1        # 1=4択フェーズ, 2=説明フェーズ（省略時は1扱い）
quiz_streak: 0       # 現フェーズでの連続正解数（Phase 1の昇格用）
quiz_fail_streak: 0  # 連続不正解数（Phase 2の降格用）
```

既存の `review_streak` / `review_log` / `#要復習` タグとは独立して管理する。

---

## フェーズ定義

| フェーズ | 出題形式 | 説明 |
|---------|---------|------|
| Phase 1（4択） | 選択肢A〜Dから1つ選ぶ | 概念に慣れるための学習段階 |
| Phase 2（説明） | 概念・重要な場面・実務での扱いまで自由記述 | 人に説明できるレベルの確認 |

---

## 昇格・降格ルール

| 条件 | アクション |
|------|-----------|
| Phase 1 で `quiz_streak` が 2 になった | 次回セッションから Phase 2 に昇格 |
| Phase 2 で `quiz_fail_streak` が 2 になった | 即座に Phase 1 に降格、両ストリークをリセット |

**「次回セッションから」の意味**: 昇格条件を満たした `/quiz` 呼び出しの中では4択を継続する。次の `/quiz` 呼び出しから説明フェーズで出題する。

---

## 正解・不正解時の処理

### Phase 1（4択フェーズ）

| 結果 | `quiz_streak` | `#要復習` |
|------|-------------|----------|
| 正解 | +1（2で昇格予約） | 変化なし |
| 不正解 | 0 にリセット | **付けない**（まだ学習中のため） |

### Phase 2（説明フェーズ）

| 結果 | `quiz_fail_streak` | `#要復習` |
|------|-------------------|----------|
| 正解 | 0 にリセット | あれば**削除** |
| 不正解 | +1（2で降格） | **追加** |

---

## `#要復習` との関係

- `#要復習` は「一度マスターしたのに忘れた」状態を示すタグ
- Phase 1 は「まだ学習中」なので不正解でもタグを付けない
- Phase 2 の正解1回で削除（4択2回＋説明1回の合計3回正解が定着の証拠）
- 従来の「日をまたいで2回連続正解」条件は廃止する

---

## 4択選択肢の生成方法

- 正解: 対象ノートの Summary を使用
- 誤答: 他のノートの Summary から近い概念を優先して引用（不足時は Claude が即興生成）
- 選択肢は A〜D の4択、順序はランダム

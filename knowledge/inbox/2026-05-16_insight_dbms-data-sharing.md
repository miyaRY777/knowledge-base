#

- [ ] `DBMS`とは？

👉 **データベースを管理・操作するためのソフトウェア**

**解説：**
`DBMS` は、データベースに保存されたデータを追加・検索・更新・削除できるようにするソフトウェアです。
データの整合性を守ったり、バックアップやアクセス権限の管理を行ったりする役割もあります。
代表例には MySQL、PostgreSQL、Oracle Database などがあります。

**具体例：**
- Railsアプリが PostgreSQL にユーザー情報を保存する
- 管理画面からデータを検索・更新する
- 複数のユーザーが同じデータを安全に使えるようにする

この例では、データベースを安全かつ効率的に扱うために `DBMS` が関係しています。

参考: [https://school.runteq.jp/v3/books/2/chapter/12](https://school.runteq.jp/v3/books/2/chapter/12)

- [ ] DBにおける`リアルタイム更新と共有`はどのような仕組み？

👉 データベース上のデータが更新されたとき、その変更を他のユーザーやアプリケーションもすぐに参照できる仕組み

**解説：**
`リアルタイム更新と共有` は、データベース上のデータが更新されたとき、その変更を他のユーザーやアプリケーションもすぐに参照できる仕組みです。
これにより、古い情報を見て作業してしまうリスクを減らせます。

**具体例：**
- 営業担当が顧客の住所を更新する
- サポート担当がすぐに最新の住所を確認できる
- 別の画面やアプリでも同じ最新情報を使える

この例では、最新の情報を複数の人が共有するために `リアルタイム更新と共有` が関係しています。

参考: [https://school.runteq.jp/v3/books/2/chapter/12](https://school.runteq.jp/v3/books/2/chapter/12)

- [ ] `中央集権型管理`とは？

👉 **データを一か所に集めて管理する考え方**

**解説：**
`中央集権型管理` は、データをバラバラの場所ではなく、データベースなどの一か所にまとめて管理する方法です。
データの重複や矛盾を減らし、利用者が信頼できる同じデータを参照しやすくなります。

**具体例：**
- 顧客情報をExcelファイルごとに分けず、1つのDBで管理する
- 商品情報をWebアプリ・管理画面・分析ツールで共通利用する
- データの修正はDB上の1か所で行う

この例では、データの重複や矛盾を防ぐために `中央集権型管理` が関係しています。

参考: [https://school.runteq.jp/v3/books/2/chapter/12](https://school.runteq.jp/v3/books/2/chapter/12)

- [ ] `複数のアプリケーションからのアクセス`のメリットは？

👉 **同じデータベースを、複数のアプリやシステムから利用できること**

**解説：**
`複数のアプリケーションからのアクセス` は、Webアプリ、スマホアプリ、管理画面などが同じデータベースを使える状態を指します。
同じDBを共有することで、アプリごとに別々のデータを持たず、一貫したデータ管理がしやすくなります。

**具体例：**
- Webアプリで登録したユーザー情報を、スマホアプリでも使う
- 管理画面で変更した商品情報が、ECサイトにも反映される
- 分析ツールが本番DBのデータを参照して売上を集計する

この例では、複数のアプリで同じデータを一貫して使うために `複数のアプリケーションからのアクセス` が関係しています。

参考: [https://school.runteq.jp/v3/books/2/chapter/12](https://school.runteq.jp/v3/books/2/chapter/12)

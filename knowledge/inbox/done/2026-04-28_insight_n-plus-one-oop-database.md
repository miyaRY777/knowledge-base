- [ ] `N+1`とは？

👉 **関連データを取るときに、必要以上にSQLが増えてしまう問題**

**解説：**
`N+1` は、一覧データを1回取得したあと、それぞれの要素ごとに関連データを追加で取りに行ってしまう状態です。
Rails Guides では、関連データの先読みをしないと、たとえば10件の一覧に対して 1回 + 10回 のようにクエリが増える例が説明されています。
対策としては、`includes`、`preload`、`eager_load` などで関連を先に読み込む方法があります。 

```
books = Book.includes(:author).limit(10)

books.each do |book|
  puts book.author.name
end
```

このコードでは、書籍ごとに著者を1件ずつ取りに行かないように、関連データを先に読み込むために `N+1` の対策をしています。
- [ ] `オブジェクト指向`とは？

👉 **データと処理をまとめて「モノ（オブジェクト）」として扱う考え方**

**解説：**
`オブジェクト指向` は、データ（状態）とそのデータに対する処理（振る舞い）をひとつにまとめて扱う設計の考え方です。
RubyやRailsでは、クラスを定義してインスタンス（オブジェクト）を作り、役割ごとに責務を分けて実装します。

**具体例：**
- `User` というクラスに「名前・メール」と「ログインする処理」をまとめる
- `Post` というクラスに「タイトル・内容」と「公開する処理」をまとめる

この例では、データと処理をひとまとまりにして管理するために `オブジェクト指向` が関係しています。

- [ ] `データベース`とは？

👉 **アプリのデータを保存・管理する場所**

**解説：**
`データベース` は、ユーザー情報や投稿などのデータを保存して、あとから取り出したり更新したりするための仕組みです。
Railsでは主に PostgreSQL や MySQL などを使い、Active Record を通して操作します

具体例：

```sql
SELECT * FROM users;
```

このコードでは、ユーザーのデータを取得するために `データベース` を使っています。}

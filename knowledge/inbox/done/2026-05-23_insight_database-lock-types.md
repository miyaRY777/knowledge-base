* [ ] `排他ロック（Exclusive Lock）`とは？

👉 **データを「自分だけが変更できる状態」にする仕組み**

**解説：**
`排他ロック` は、ある処理がデータを更新している間、他の処理が同じデータを変更できないようにする仕組みです。
同時更新によるデータのズレや上書きを防ぐために使います。
銀行の残高更新や在庫数の変更など、データ整合性が重要な場面でよく使われます。

```ruby
User.transaction do
  user = User.lock.find(1)
  user.points -= 100
  user.save!
end
```

このコードでは、他の処理から同じユーザー情報を同時更新されないようにするために `排他ロック` を使っています。

* [ ] `共有ロック（Shared Lock）`とは？

👉 **複数人で読み取りはできるが、変更は制限する仕組み**

**解説：**
`共有ロック` は、データを読むことは許可しつつ、他の処理による変更を防ぐための仕組みです。
複数の処理が同時にデータを参照できるため、読み取り中心の処理で使われます。
ただし、共有ロック中は更新系の処理が待たされる場合があります。

**具体例：**

* AさんとBさんが同じデータを閲覧する
* 閲覧中は「内容を書き換える処理」を止める
* 読み取りが終わったら更新可能になる

この例では、データを安全に参照するために `共有ロック` が関係しています。

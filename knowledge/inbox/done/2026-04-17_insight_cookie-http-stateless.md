{
- [ ] `Cookie`とは？

👉 **ブラウザに保存される、ユーザー情報の小さなデータのこと**

**解説：**
Cookieは、ユーザーの状態（ログイン情報や設定など）をブラウザに保存する仕組みです。
次回アクセス時にその情報を使うことで、ログイン状態の維持や表示のカスタマイズができます。

具体例：

```http
Set-Cookie: user_id=12345; path=/
```

このコードでは、サーバーがユーザーIDをブラウザに保存させるために Cookie を使っています。

---

- [ ] `HTTP（Hypertext Transfer Protocol）`とは？

👉 **Webブラウザとサーバーが通信するためのルール**

**解説：**
HTTPは、ブラウザとサーバーの間でデータをやり取りするためのプロトコルです。
Webページの表示やフォーム送信など、すべての通信の基本になります。

具体例：

```http
GET /index.html HTTP/1.1
Host: example.com
```

このコードでは、ブラウザがサーバーにページをリクエストするために HTTP を使っています。

---

- [ ] `ステートレス（stateless）`とは？

👉 **前の状態を覚えない仕組み**

**解説：**
ステートレスとは、リクエストごとに状態を保持しない性質のことです。
HTTPは毎回の通信が独立しているため、ログイン状態などは自動では保持されません。

具体例：

```text
1回目のリクエスト → ログイン
2回目のリクエスト → ログイン状態は覚えていない
```

このコードでは、HTTPが状態を保持しないことを説明するために ステートレス を使っています。

---

- [ ] `Cookieの役割・必要な理由`とは？

👉 **ステートレスなHTTPでユーザーの状態を保持し、便利なWeb体験を実現する仕組み**

**解説：**
HTTPは状態を保持しないため、そのままだとログイン状態やカート情報は維持できません。
Cookieを使うことで、ユーザー情報（認証・行動履歴・設定など）を保存し、ログイン維持やカート保持、コンテンツ最適化などが可能になります。
ただし、個人情報やプライバシーの扱いには注意が必要です。

具体例：

```http
Set-Cookie: user_id=12345
Cookie: user_id=12345
```

このコードでは、ユーザー情報を保存して次回のリクエストでも識別できるようにするために Cookie を使っています。

Cookieは主に以下のような目的で使用されます：
1. **ユーザーの認証情報を保存**:
   - ユーザーがWebサイトにログインした場合、その情報をCookieに保存することで、次回の訪問時に再度ログインする手間を省くことができます。これにより、ユーザーは毎回ユーザー名やパスワードを入力する必要がなくなります。
2. **サイトの利用状況を追跡**:
   - Cookieを使って、ユーザーがどのページを訪れたか、どのような操作を行ったかを記録することができます。これにより、Webサイトがユーザーの行動を分析し、より良いサービスを提供するための改善点を見つけることができます。
3. **ショッピングカートの情報を保存**:
   - ECサイトなどでは、ユーザーがショッピングカートに追加したアイテムの情報をCookieに保存することがあります。これにより、ユーザーがWebサイトを離れた後でも、再訪問時にカートの内容が保存されていることが確認できます。
4. **個別化されたコンテンツの提供**:
   - ユーザーが過去に閲覧したページや設定した内容に基づいて、Webサイトがユーザーに最適なコンテンツや広告を表示することができます。たとえば、ユーザーの興味に合わせたおすすめ商品やサービスを表示する際に使用されます。
- [ ] `cookies[:user_id] = current_user.id`とは？

👉 **ユーザーIDをCookieに保存するコード**

**解説：**
`cookies` は、ブラウザにデータを保存するためのRailsの仕組みです。
このコードは、ログインしているユーザーのIDをブラウザに保存し、次回のリクエストでもそのユーザーを識別できるようにするために使います。
ただし、Cookieはユーザー側に保存されるため、そのまま使うのは安全面に注意が必要です。

具体例：

```
cookies[:user_id] = current_user.id
```

このコードでは、ログインしているユーザーを識別できるように、ユーザーIDをCookieに保存するために cookies を使っています。

補足：

Rails の `cookies` は、コントローラから Cookie を読み書きするための仕組みで、書き込むとレスポンスの `Set-Cookie` ヘッダーとして返され、次回以降のリクエストでブラウザから送られます。}

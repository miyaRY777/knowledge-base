データベースサーバーとは？

👉 データを保存・管理して、必要なときに取り出せるようにするサーバー

解説：
データベースサーバー は、アプリのデータを保存し、検索・更新・削除できるようにするサーバーです。
Webアプリでは、ユーザー情報や投稿データなどを管理するためによく使われます。
代表例として、PostgreSQL や MySQL があります。

具体例：
ECサイトで会員登録すると、ユーザー情報が保存される

ログイン時に、メールアドレスとパスワードを検索する

商品一覧を表示するときに、商品データを取得する

この例では、アプリのデータを安全に保存・取得するために データベースサーバー が関係しています。

参考: PostgreSQL公式サイト MySQL公式サイト


分散データベースとは？

👉 複数のサーバーに分けてデータを管理するデータベース方式

解説：
分散データベース は、1台ではなく複数のサーバーにデータを分散して保存・管理する仕組みです。
大量アクセスへの対応や、障害時にサービスを止まりにくくするために使われます。
ただし、データの同期や整合性管理が複雑になる点に注意が必要です。

具体例：
日本とアメリカにサーバーを配置する

日本ユーザーは日本サーバーに接続する

アメリカユーザーはアメリカサーバーに接続する

どちらかのサーバーが故障しても、別サーバーで継続できる場合がある

この例では、負荷分散や可用性向上のために 分散データベース が関係しています。

参考: Google Cloud Spanner公式説明 Apache Cassandra公式サイト

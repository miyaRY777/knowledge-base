{* [ ] `トップレベルドメイン（TLD）`とは？  👉 **ドメイン名の一番右側にある部分**  **解説：** `トップレベルドメイン（TLD）` は、ドメイン名をドットで区切ったときの一番右側の部分です。 たとえば `example.com` の `com`、`example.jp` の `jp` がトップレベルドメインです。 ドメイン名は右側ほど上位の階層になります。 参考: [JPRS TLD（トップレベルドメイン）](https://jprs.jp/glossary/index.php?ID=0058) / [JPNIC ドメイン名のしくみ](https://www.nic.ad.jp/ja/dom/system.html)  **具体例：**  * `example.com` の `com` * `example.jp` の `jp` * `example.org` の `org`  この例では、ドメイン名の一番右側で大きな分類を表すために `トップレベルドメイン（TLD）` が関係しています。  ---  * [ ] `セカンドレベルドメイン`とは？  👉 **トップレベルドメインの左側にある部分**  **解説：** `セカンドレベルドメイン` は、トップレベルドメインの1つ左にある部分です。 たとえば `example.com` なら `example` がセカンドレベルドメインです。 ただし、`example.co.jp` のようなドメインでは、`co` がセカンドレベルドメイン、`example` はサードレベルドメインになります。 参考: [JPNIC ドメイン名のしくみ](https://www.nic.ad.jp/ja/dom/system.html) / [ICANN About Domain Names](https://www.icann.org/resources/pages/about-domain-names-2018-08-30-en)  **具体例：**  * `example.com` の `example` * `google.com` の `google` * `jprs.co.jp` の `co`  この例では、トップレベルドメインのすぐ左の階層を表すために `セカンドレベルドメイン` が関係しています。  ---  * [ ] `サブドメイン`とは？  👉 **メインのドメインをさらに分けて使うための名前**  **解説：** `サブドメイン` は、取得したドメインの左側に名前を追加して、用途ごとに分けるために使います。 たとえば `blog.example.com` の `blog` や、`app.example.com` の `app` がサブドメインとして使われます。 Webアプリでは、本番環境・管理画面・ブログなどを分けたいときに使われることがあります。 参考: [ICANN About Domain Names](https://www.icann.org/resources/pages/about-domain-names-2018-08-30-en)  **具体例：**  * `blog.example.com`：ブログ用 * `app.example.com`：アプリ用 * `admin.example.com`：管理画面用  この例では、同じドメインの中で用途ごとに場所を分けるために `サブドメイン` が関係しています。 }

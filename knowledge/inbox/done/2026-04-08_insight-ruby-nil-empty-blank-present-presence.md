`nil?`とは？
 👉 **値が **`nil`** そのものかどうかを調べるメソッド**
 **解説：** `nil?` は、その値が本当に `nil` のときだけ `true` になります。
 空文字 `""` や空配列 `[]` は `nil` ではないので、`true` にはなりません。
 「値そのものがないか」をはっきり確認したいときに使います。
 ```ruby
value = nil

value.nil?   # => true
"".nil?      # => false
[].nil?      # => false
```
 このコードでは、その値が `nil` そのものかどうかを確認するために `nil?` を使っています。
 ---
 - [ ] `empty?`とは？
 👉 **文字列や配列などの中身が空かどうかを調べるメソッド**
 **解説：** `empty?` は、文字列・配列・ハッシュなどの「中身を持つオブジェクト」に使います。
 ただし、`nil` には使えず、`nil.empty?` はエラーになります。
 そのため、`nil` の可能性がある値にいきなり使うと危険です。
 ```ruby
"".empty?      # => true
"abc".empty?   # => false
[].empty?      # => true
{}.empty?      # => true
```
 このコードでは、文字列や配列やハッシュの中身が空かどうかを確認するために `empty?` を使っています。
 ---
 - [ ] `blank?`とは？
 👉 `nil`** や空文字などを、まとめて「空っぽ」と判定できるRailsのメソッド**
 **解説：** `blank?` は Rails の Active Support が追加しているメソッドです。
 `nil`、`""`、`" "`、`[]`、`{}`、`false` などをまとめて空扱いできます。
 `nil` の可能性がある値にも使いやすいので、Railsではよく使われます。
 ```ruby
nil.blank?      # => true
"".blank?       # => true
"   ".blank?    # => true
[].blank?       # => true
false.blank?    # => true
0.blank?        # => false
```
 このコードでは、実質的に中身がない値かどうかをまとめて確認するために `blank?` を使っています。
 ---
 - [ ] `present?`とは？
 👉 `blank?`** の反対で、「中身がある」と言えるかを調べるメソッド**
 **解説：** `present?` は `blank?` が `false` のときに `true` になります。
 「空ではないなら使いたい」と書きたいときに便利です。
 これも Rails の Active Support のメソッドです。
 ```ruby
name = "Ruby"

name.present?   # => true
"".present?     # => false
nil.present?    # => false
```
 このコードでは、値が空ではなく、使える中身を持っているかどうかを確認するために `present?` を使っています。
 ---
 - [ ] `presence`とは？
 👉 **値が空でなければその値を返し、空っぽなら **`nil`** を返すRailsのメソッド**
 **解説：** `presence` は、「使える値ならそのまま使い、空なら `nil` にする」ときに便利です。
 そのため、フォールバック値を使いたい場面でよく使います。
 これも Rails の Active Support のメソッドです。
 ```ruby
label = ""

label.presence || "この部屋の趣味" # => "この部屋の趣味"
```
 このコードでは、`label` が空なら代わりの文字を使うために `presence` を使っています。
 ---
 - [ ] `empty?`を使うときの注意点とは？
 👉 `nil`** の可能性がある値に、そのまま使うとエラーになること**
 **解説：** `empty?` は便利ですが、`nil` に対して呼ぶと `NoMethodError` になります。
 そのため、値が `nil` かもしれないなら、先に `nil?` で確認するか、Rails なら `blank?` を使うほうが安全です。
 「空文字かもしれないし、`nil` かもしれない」という場面では特に注意が必要です。
 ```ruby
value = nil

value.empty? # NoMethodError
```
 このコードでは、`nil` に対して `empty?` を呼ぶとエラーになることを確認するために `empty?` を使っています。
 ---
 - [ ] `nil` の可能性があるなら、先に `nil?` を見るとは？
 👉 `empty?`** を呼ぶ前に、その値が **`nil`** ではないことを確認すること**
 **解説：** `empty?` は `nil` に使えないので、`nil` の可能性がある値にはそのまま使えません。
 そのため、先に `nil?` で確認してから `empty?` を使う書き方があります。
 ただし Rails では、同じ目的なら `blank?` のほうが短く書けることが多いです。
 ```ruby
value = nil

if !value.nil? && value.empty?
  puts "空文字です"
end
```
 このコードでは、`nil` に `empty?` を呼んでエラーにならないように、先に `nil?` を確認するために `nil?` を使っています。
 ---
 - [ ] `blank?`が安全と言われるのはなぜですか？
 👉 `nil`** にもそのまま使えて、空文字や空白だけもまとめて判定できるから**
 **解説：** `blank?` は Rails のメソッドなので、`nil` に使ってもエラーになりません。
 さらに `""` や `" "` もまとめて空扱いできます。
 「とにかく空っぽかどうか見たい」ときに、初心者にも使いやすいメソッドです。
 ```ruby
value = nil

if value.blank?
  puts "空です"
end
```
 このコードでは、`nil` かもしれない値を安全に空判定するために `blank?` を使っています。
 ---
 - [ ] `presence`はどういうときに便利ですか？
 👉 **空なら別の値を使いたいときに便利**
 **解説：** `presence` は、「値が入っていればそれを使い、空なら別の値にしたい」ときによく使います。
 `if value.present?` のような分岐を短く書けるのも便利な点です。
 フォーム入力・部屋名・表示名などでよく使います。
 ```ruby
nickname = params[:nickname].presence || "no-name"
```
 このコードでは、入力されたニックネームが空なら `"no-name"` を使うために `presence` を使っています。

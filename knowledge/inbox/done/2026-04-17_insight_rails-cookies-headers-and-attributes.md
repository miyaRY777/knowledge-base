{
- [ ] `cookies[:user_id] = current_user.id`とは？

👉 **ユーザーIDをCookieに保存するコード**

**解説：**
`cookies` は、ブラウザにデータを保存するためのRailsの仕組みです。
このコードは、ログインしているユーザーのIDをブラウザに保存し、次回のリクエストでもそのユーザーを識別できるようにするために使います。
ただし、Cookieはユーザー側に保存されるため、そのまま使うのは安全面に注意が必要です。

具体例：

```
cookies[:user_id] = current_user.id
```

このコードでは、ログインしているユーザーを識別できるように、ユーザーIDをCookieに保存するために cookies を使っています。

補足：

Rails の `cookies` は、コントローラから Cookie を読み書きするための仕組みで、書き込むとレスポンスの `Set-Cookie` ヘッダーとして返され、次回以降のリクエストでブラウザから送られます。

`Set-Cookieヘッダー`とは？

👉 **サーバーがブラウザにCookieを保存させるためのHTTPヘッダー**

**解説：**
`Set-Cookie` はHTTPレスポンスで使われ、ブラウザに「このデータを保存して」と指示します。
ログイン後にユーザーIDやセッションIDを保存するときによく使われます。

```http
Set-Cookie: session_id=abc123; path=/; HttpOnly
```

このコードでは、ブラウザに session_id を保存させるために Set-Cookieヘッダー を使っています。

---

- [ ] `Cookieヘッダー`とは？

👉 **ブラウザがサーバーにCookieを送るときに使うHTTPヘッダー**

**解説：**
`Cookie` ヘッダーは、保存されているCookieをサーバーに送り返すときに使います。
これにより、サーバーはユーザーを識別できます。

```http
GET /mypage HTTP/1.1
Cookie: session_id=abc123
```

このコードでは、保存されている session_id をサーバーに送るために Cookieヘッダー を使っています。

---

- [ ] `expires属性`とは？

👉 **Cookieの有効期限を指定するための設定**

**解説：**
`expires` を指定すると、その日時までCookieが保存されます。
指定しない場合は、ブラウザを閉じると削除されることがあります。

```http
Set-Cookie: session_id=abc123; expires=Wed, 21 Oct 2026 07:28:00 GMT
```

このコードでは、指定した日時までCookieを保持するために expires属性 を使っています。

---

- [ ] `HttpOnly属性`とは？

👉 **JavaScriptからCookieを参照できなくするための設定**

**解説：**
`HttpOnly` を付けると、JavaScriptからCookieを取得できなくなります。
XSS攻撃によるCookieの盗難を防ぐために使われます。

```http
Set-Cookie: session_id=abc123; HttpOnly
```

このコードでは、JavaScriptからアクセスできないようにするために HttpOnly属性 を使っています。

---

- [ ] `Secure属性`とは？

👉 **HTTPS通信のときだけCookieを送信するための設定**

**解説：**
`Secure` を付けると、HTTPS通信のときだけCookieが送信されます。
これにより、通信中の盗聴リスクを減らせます。

```http
Set-Cookie: session_id=abc123; Secure
```

このコードでは、安全な通信時のみ送信するために Secure属性 を使っています。
- [ ] `Cookieの仕組み`とは？

👉 **サーバーとブラウザがCookieをやり取りしてユーザー情報を保持する仕組み**

**解説：**
Cookieは、サーバーがブラウザに送った情報をブラウザが保存し、次回のリクエストでその情報をサーバーに送り返す仕組みです。
HTTPはステートレスなので、そのままではログイン状態などを覚えられません。
そのため、Cookieを使ってユーザーの状態を管理します。
注意点として、パスワードのような機密情報をそのままCookieに保存するのは避ける必要があります。

具体例：

**Cookieの動作の流れ**
1. **ユーザーがWebサイトにアクセスする**
   - まず、ユーザーがブラウザでWebサイトを開きます。
   - この時点では、サーバーはまだ「誰からのアクセスか」を覚えていないことがあります。
2. **サーバーがCookieをブラウザに送信する**
   - サーバーはHTTPレスポンスの `Set-Cookie` ヘッダーを使って、ブラウザにCookieを送ります。
   - たとえば、ログイン後に「このセッションは session_id=abc123」と識別するための情報を送ることがあります。

```http
Set-Cookie: session_id=abc123; expires=Wed, 21 Oct 2026 07:28:00 GMT; path=/; domain=example.com; secure; HttpOnly
```

1. **ブラウザがCookieを保存する**
   - ブラウザは、サーバーから送られてきたCookieを保存します。
   - `expires` が指定されていれば、その期限まで保存されます。
   - 保存場所や保持のされ方はブラウザに依存します。
2. **ユーザーが同じWebサイトに再アクセスする**
   - ユーザーがもう一度そのWebサイトを開くと、ブラウザは保存していたCookieを確認します。
   - そのCookieが対象のサイト用のものであれば、次のリクエストに含めます。
3. **ブラウザがCookieをサーバーに送信する**
   - ブラウザはHTTPリクエストの `Cookie` ヘッダーに、保存していたCookieを入れて送信します。

```http
Cookie: session_id=abc123
```

1. **サーバーがCookieを読み取る**
   - サーバーは受け取ったCookieを見て、「このリクエストは session_id=abc123 のセッションからだな」と判断します。
   - これにより、ログイン済みユーザーとして画面を表示したり、ユーザーごとの情報を返したりできます。
2. **ユーザーに合ったデータを返す**
   - サーバーはCookieの内容をもとに、マイページやカート情報など、そのユーザーに合ったデータを返します。
   - このようにして、Webサイトは状態を持っているように見せることができます。

```http
# ① サーバー → ブラウザ
Set-Cookie: session_id=abc123; path=/; HttpOnly

# ② ブラウザが保存

# ③ 次回アクセス時に送信
Cookie: session_id=abc123
```

このコードでは、ユーザーを識別してログイン状態を維持するために Cookieの仕組み を使っています。}

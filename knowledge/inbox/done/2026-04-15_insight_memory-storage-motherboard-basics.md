{# 

- [ ] `主記憶装置（メモリ / RAM）`とは？

👉 **CPUが直接使う、一時的な作業場所となる記憶装置**

**解説：**
主記憶装置は、実行中のプログラムや処理中のデータを一時的に保存するための場所です。CPUが直接アクセスできるため高速ですが、電源を切ると内容は消えます（揮発性）。

具体例：

```text
1. ストレージからプログラムを読み込む
2. メモリ上に展開する
3. CPUがメモリのデータを使って処理する
```

このコードでは、CPUが処理を行うためにメモリ上のデータを使う流れを説明するために 主記憶装置 を使っています。

---

- [ ] `補助記憶装置（ストレージ）`とは？

👉 **データを長期間保存するための記憶装置**

**解説：**
補助記憶装置は、ファイルやプログラムなどを保存しておく場所です。HDDやSSDが該当し、電源を切ってもデータが消えない（不揮発性）という特徴があります。ただし、メモリよりアクセス速度は遅いです。

具体例：

```text
保存: ファイルをストレージに書き込む
起動: ストレージからメモリへ読み込む
```

このコードでは、データを保存する場所と、実行時に読み込む流れを説明するために 補助記憶装置 を使っています。

---

- [ ] `主記憶装置と補助記憶装置の違い`とは？

👉 **「速さと一時性」と「保存性と容量」の違い**

**解説：**
主記憶装置は高速でCPUが直接使う作業領域、補助記憶装置は低速だが長期保存できる保管場所です。実際の処理では、ストレージからメモリに読み込んでからCPUが実行します。

具体例：

```text
ストレージ → メモリ → CPU
```

このコードでは、コンピュータが処理を行う基本的な流れを理解するために 主記憶装置と補助記憶装置の違い を使っています。
- [ ] `マザーボード（Motherboard）`とは？

👉 **コンピュータのすべての部品をつなぐ土台となる基盤**

**解説：**
マザーボードは、CPU・メモリ・ストレージなどの部品を取り付けて、互いに通信できるようにする基盤です。データのやり取り（バス）や電力の供給も担っており、コンピュータ全体をつなぐ役割を持ちます。

具体例：

```text
CPU ─┐
     ├─ マザーボード ─ ストレージ
メモリ ┘
```

このコードでは、各パーツがマザーボードを通して接続されている構造を理解するために マザーボード を使っています。

---

- [ ] `マザーボードにおけるバス（Bus）`とは？

👉 **部品同士でデータをやり取りするための通り道**

**解説：**
バスは、CPU・メモリ・ストレージなどの間でデータを転送する経路です。マザーボード上にあり、これによって各部品が連携して動作します。

具体例：

```text
CPU → バス → メモリ
```

このコードでは、データが部品間を移動する仕組みを理解するために バス を使っています。

---

- [ ] マザーボードにおける`スロット / ソケット`とは？

👉 **部品を取り付けるための接続口**

**解説：**
スロットやソケットは、CPUやメモリなどを物理的に取り付ける場所です。種類が決まっているため、対応していない部品は取り付けできない点に注意が必要です。

具体例：

```text
CPU → CPUソケットに装着
メモリ → メモリスロットに装着
```

このコードでは、各パーツが決められた場所に取り付けられることを理解するために スロット / ソケット を使っています。}

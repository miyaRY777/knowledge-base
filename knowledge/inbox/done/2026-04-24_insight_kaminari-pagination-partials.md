- [ ] `_paginator.html.erb`とは？

👉 **ページネーション全体のレイアウトを定義するビューのこと**

**解説：**
Kaminariでページネーションを表示するときの「全体の枠」を決めるテンプレートです。
ページ番号や前後ボタンをどの順番で並べるかをここで定義します。

具体例：

```erb
<nav>
  <%= paginator.render do %>
    <%= prev_page_tag %>
    <%= each_page do |page| %>
      <%= page_tag page %>
    <% end %>
    <%= next_page_tag %>
  <% end %>
</nav>
```

このコードでは、ページネーションの全体構造を定義するために `_paginator.html.erb` を使っています。

- [ ] `_page.html.erb`とは？

👉 **ページ番号ボタンを表示するためのビュー**

**解説：**
「1」「2」「3」などのページ番号リンク1つ分の見た目を定義します。
現在のページかどうかで見た目を変えることもできます。

具体例：

```erb
<%= link_to page, url, class: "page-link" %>
```

このコードでは、ページ番号のリンクを表示するために `_page.html.erb` を使っています。

- [ ] `render`とは？

👉 **ビューやパーシャルを組み合わせてHTMLを作るためのメソッド**

**解説：**
`render` は、別のビューやパーシャルを呼び出して、最終的な画面を組み立てるために使います。
1リクエスト内で何度も使われ、レイアウト・ビュー・パーシャルが組み合わさって1つのHTMLになります。
ただし、コントローラでは `render` は基本1回しか呼べず、複数回呼ぶとエラーになります。

具体例：

```
<%= render "shared/header" %>
<%= render "posts/post", post: @post %>
<%= render "shared/footer" %>
```

このコードでは、複数のパーツを組み合わせて1つの画面を作るために render を使っています。}

`メモリ`とは？

👉 **プログラムが動いている間、データを一時的に置いておく作業スペース**

**解説：**
`メモリ` は、プログラムが今すぐ使うデータを置いておく場所です。
変数、オブジェクト、取得したデータなどが一時的に置かれます。
容量には限りがあるため、大量のデータを一度に載せすぎると処理が重くなる原因になります。

**具体例：**
- Railsで `User.all` を実行する
- ユーザー全件のデータがメモリ上に読み込まれる
- 件数が多いと、メモリをたくさん使う

この例では、取得したデータを一時的に置くために `メモリ` が関係しています。

- [ ] `ディスク`とは？

👉 **データを長期的に保存しておく場所**

**解説：**
`ディスク` は、ファイルやデータベースのデータなどを保存しておく場所です。
メモリより多くのデータを保存できますが、読み書きの速度はメモリより遅いです。
そのため、処理中のデータはできるだけメモリで扱い、必要に応じてディスクから読み込みます。

**具体例：**
- データベースのデータはディスクに保存されている
- RailsがSQLを実行して、必要なデータを取り出す
- 取り出したデータをメモリに載せて処理する

まとめ：

HDDやSSDなど、データを長期保存するためのストレージのこと

- [ ] `静的領域`とは？

👉 **プログラムの実行中、長く存在するデータを置くメモリ領域**

**解説：**
`静的領域` は、プログラム起動中に長く使われる情報を置く場所です。
クラスやメソッドの情報など、実行中ずっと必要になるものが置かれることがあります。
細かい管理方法は言語や実行環境によって異なります。

**具体例：**
- Rubyプログラムを起動する
- クラスやメソッドの情報が読み込まれる
- プログラム実行中、それらの情報が使われ続ける

- [ ] `ヒープ`とは？

👉 **動的に作られたオブジェクトを置くメモリ領域**

**解説：**
`ヒープ` は、プログラムの実行中に作られるオブジェクトを置く場所です。
RubyやRailsでは、モデルのインスタンスや文字列、配列など多くのオブジェクトがヒープに置かれます。
使われなくなったオブジェクトは、GCによって回収されます。

```ruby
users = User.all.to_a
```

このコードでは、取得したユーザー情報をオブジェクトとしてメモリに載せるために `ヒープ` が関係しています。

- [ ] `スタック`とは？

👉 **メソッド呼び出し中の一時的な情報を置くメモリ領域**

**解説：**
`スタック` は、今実行しているメソッドのローカル変数や引数などを管理する場所です。
メソッドが呼ばれると情報が積まれ、メソッドが終わると自動的に取り除かれます。
一時的な処理の流れを管理する場所と考えるとわかりやすいです。

```ruby
def greet(name)
  message = "こんにちは、#{name}さん"
  puts message
end

greet("田中")
```

このコードでは、`name` や `message` のようなメソッド内の一時的な情報を扱うために `スタック` が関係しています。

- [ ] `メモリを使いすぎるデメリット` について

👉 **アプリが遅くなったり、最悪の場合はプロセスが落ちたりすること**

**解説：**

メモリを使いすぎると、処理が重くなる、スワップでさらに遅くなる、OOMでプロセスが落ちるなどです。
特にRailsで大量データを一度に読み込むと、メモリを圧迫しやすくなります。
大量データを扱う処理では、一度に読み込む量を減らすことが大切です。

**具体例：**
- `User.all` で100万件を一度に読み込む
- メモリ使用量が大きくなる
- GCが頻繁に動いて処理が遅くなる
- 物理メモリが足りないとスワップが発生する
- 最悪の場合、OOMでプロセスが強制終了される

- [ ] `GC（ガベージコレクション）`とは？

👉 **使われなくなったオブジェクトを自動で片付ける仕組み**

**解説：**
`GC（ガベージコレクション）` は、もう使われていないメモリ上のオブジェクトを自動で回収する仕組みです。
RubyではGCがあるため、開発者が手動でメモリ解放を書く場面は多くありません。
ただし、メモリを大量に使うとGCが頻繁に走り、処理が遅くなる場合があります。

**具体例：**
- 大量のActiveRecordオブジェクトを作る
- 不要になったオブジェクトが増える
- GCがそれらを回収する
- GC中はアプリの処理が一時的に止まることがある

- [ ] `スワップ`とは？

👉 **メモリが足りないときに、ディスクを一時的にメモリ代わりに使う仕組み**

**解説：**
`スワップ` は、物理メモリが足りなくなったときに、OSがディスクの一部をメモリの代わりに使う仕組みです。
ただし、ディスクはメモリより遅いため、スワップが増えると処理がかなり遅くなることがあります。
Railsアプリでも大量データ処理でメモリを使いすぎると、スワップが問題になる場合があります。

**具体例：**
- Railsのバッチ処理で大量データを一気に読み込む
- 物理メモリが足りなくなる
- OSがディスクをメモリ代わりに使う
- ディスクアクセスが増えて処理が激遅になる

---

- [ ] `OOM`とは？

👉 **メモリ不足によってプロセスが強制終了される状態**

**解説：**
`OOM` は `Out Of Memory` の略で、使えるメモリが足りなくなった状態です。
OSや実行環境によっては、メモリを使いすぎたプロセスが強制終了されることがあります。
Railsでは、大量データを一括処理するバッチなどで注意が必要です。

**具体例：**
- 大量の画像データやユーザーデータを一度に処理する
- メモリを使い切る
- OSがプロセスを強制終了する
- バッチ処理やサーバープロセスが途中で落ちる

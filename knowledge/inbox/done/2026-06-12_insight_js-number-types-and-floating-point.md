* [ ] `整数型`とは？

👉 **小数点を含まない整数を扱うデータ型**

**解説：**
`整数型`は、`10`、`0`、`-25`のような整数を保存・計算するために使います。
多くの言語では使用するビット数が決まっているため、扱える値の範囲にも上限と下限があります。

```js
const count = 232;

console.log(count);
console.log(typeof count);
```

このコードでは、整数のように見える`232`を扱っています。ただし、JavaScriptでは整数専用の型ではなく、通常は`number`型として扱われます。

* [ ] `整数型の値の範囲`とは？

👉 **整数型で保存できる最小値から最大値までの範囲**

**解説：**
コンピュータのメモリには限りがあるため、固定されたビット数の整数型では、すべての整数を保存できるわけではありません。
範囲を超えたときの動作は、言語やデータ型によって異なります。

**具体例：**

* 符号付き32ビット整数では、一般的に`-2,147,483,648`から`2,147,483,647`までを表せる
* それより大きな整数には、64ビット整数など、より広い範囲の型を使う
* 範囲を超えると、エラーや意図しない値になる場合がある

この例では、保存する整数に適したデータ型を選ぶために`整数型の値の範囲`が関係しています。

* [ ] `符号付き整数`とは？

👉 **正の数、負の数、0を扱える整数の表現方法**

**解説：**
`符号付き整数`は、数値が正か負かを区別できる整数です。
一般的な32ビット符号付き整数では、負の数を含めて約43億通りの値を表現します。

**具体例：**

* `100`は正の整数
* `-100`は負の整数
* `0`は正でも負でもない整数
* 32ビット符号付き整数では、一般に`-2³¹`から`2³¹ - 1`までを扱う

この例では、負の数も保存するために`符号付き整数`が使われています。

* [ ] `32ビット整数`とは？

👉 **32個の0と1を使って整数を表現するデータ形式**

**解説：**
32ビットでは、`2³²`通りのビットパターンを作れます。
符号付き整数として使う場合は、一般的に`-2³¹`から`2³¹ - 1`までを表現します。

**具体例：**

* 32ビットでは`4,294,967,296`通りのビットパターンがある
* 符号付きの場合は、負の数と正の数に範囲を分ける
* Javaの`int`は32ビット符号付き整数として定義されている
* C++の`int`のビット数は処理系によって異なるが、多くの環境では32ビット

この例では、整数をどれだけ広い範囲で保存できるかを決めるために`32ビット整数`が関係しています。

* [ ] `浮動小数点型`とは？

👉 **小数や非常に大きい・小さい数を近似して扱うデータ型**

**解説：**
`浮動小数点型`は、`3.14`や`0.001`などの小数を扱うために使います。
広い範囲の数値を表現できますが、すべての小数を正確に保存できるわけではありません。

```js
const price = 3.14;

```

このコードでは、小数の`3.14`を扱うためにJavaScriptの`number`型を使っています。

* [ ] `float型`とは？

👉 **主に32ビットで小数を扱う浮動小数点型**

**解説：**
`float型`は、小数を比較的少ないメモリで扱うためのデータ型です。
一般的なIEEE 754の単精度形式では32ビットを使用しますが、正確な仕様は言語や環境によって異なる場合があります。

```java
float rate = 0.5f;

System.out.println(rate);
```

このコードでは、小数の`0.5`を保存するために`float型`を使っています。

多くの環境では、IEEE 754の倍精度64ビット浮動小数点形式が使われています。

```java
double price = 19.99;

System.out.println(price);
```

このコードでは、小数の`19.99`を比較的高い精度で保存するために`double型`を使っています。

* [ ] `浮動小数点誤差`とは？

👉 **小数を有限のビット数で近似して保存することで発生する誤差**

**解説：**
コンピュータは多くの小数を2進数で正確に表せないため、表現可能な近い値へ丸めて保存します。
そのため、単純な小数計算でも期待した結果とわずかに異なる場合があります。

```js
console.log(0.1 + 0.2);
// 0.30000000000000004
```

このコードでは、`0.1`と`0.2`を正確な2進数として保存できないために`浮動小数点誤差`が発生しています。

* [ ] `2進小数`とは？

👉 **小数部分を2のマイナス乗で表した数**

**解説：**
`2進小数`では、小数点以下の各桁が`2⁻¹`、`2⁻²`、`2⁻³`のような値を表します。
コンピュータは内部で小数を扱うときに、この形式を利用します。

**具体例：**

* 2進数の`0.011`を考える
* `0 × 2⁻¹ + 1 × 2⁻² + 1 × 2⁻³`
* `0 + 0.25 + 0.125`
* 10進数では`0.375`になる

この例では、10進数の`0.375`をコンピュータ内部で表現するために`2進小数`が関係しています。

* [ ] `10進小数を2進小数へ変換する方法`とは？

👉 **小数部分に2を掛け、整数部分を順番に取り出す方法**

**解説：**
10進小数に繰り返し2を掛け、その結果の整数部分を上から順番に並べます。
小数部分が`0`になれば変換は終了しますが、数によっては同じ計算が無限に続きます。

**具体例：**

* `0.375 × 2 = 0 + 0.75`なので、最初の桁は`0`
* `0.75 × 2 = 1 + 0.5`なので、次の桁は`1`
* `0.5 × 2 = 1 + 0`なので、次の桁は`1`
* したがって、`0.375`は2進数で`0.011`

この例では、10進数の小数を2進数で表すために`10進小数を2進小数へ変換する方法`を使っています。

* [ ] `循環小数`とは？

👉 **同じ数字の並びが無限に繰り返される小数**

**解説：**
`循環小数`は、小数部分が途中で終わらず、同じパターンを繰り返す数です。
10進数では有限小数でも、2進数へ変換すると循環小数になる場合があります。

**具体例：**

* 10進数の`0.1`を2進数へ変換する
* `0.00011001100110011...`のように`0011`が繰り返される
* 有限のビット数には収まらないため、途中で丸める必要がある

この例では、`0.1`を2進数で正確に表せない理由として`循環小数`が関係しています。

* [ ] `丸め`とは？

👉 **正確に保存できない数を、表現可能な近い数へ置き換える処理**

**解説：**
コンピュータは、有限のビット数では保存できない小数を、近い値へ変換して保存します。
この処理によって、元の値との間にわずかな誤差が生じる場合があります。

**具体例：**

* `0.1`は2進数では無限に続く
* メモリには有限のビット数しか保存できない
* 途中までの値を使い、表現可能な近い数へ置き換える
* 計算すると誤差が結果に現れる場合がある

この例では、有限のメモリに小数を保存するために`丸め`が行われています。

* [ ] `2進数で正確に表せる小数`とは？

👉 **約分した分数の分母が2のべき乗になる小数**

**解説：**
10進小数を分数にして約分したとき、分母が`2`、`4`、`8`などの2のべき乗なら、有限の2進小数で表せます。
分母に2以外の素因数が残る場合は、2進数では循環小数になります。

**具体例：**

* `0.375 = 375 / 1000`
* 約分すると`3 / 8`
* `8 = 2³`なので、2進数で正確に表せる
* 実際に`0.375`は2進数で`0.011`

この例では、小数が誤差なく保存できるか判断するために`2進数で正確に表せる小数`という考え方が関係しています。

* [ ] `2進数で正確に表せない小数`とは？

👉 **約分した分数の分母を2のべき乗だけで表せない小数**

**解説：**
分母に`5`など、2以外の素因数が残る小数は、2進数では有限桁で表せません。
そのため、浮動小数点型では近い値として保存されます。

**具体例：**

* `0.1 = 1 / 10`
* `10 = 2 × 5`
* 分母に`5`が含まれている
* 2進数では`0.000110011...`と無限に続く

この例では、`0.1`を正確な2進小数として保存できないために`2進数で正確に表せない小数`が関係しています。

* [ ] `小数の比較`とは？

👉 **浮動小数点誤差を考慮して小数同士を比べること**

**解説：**
計算した小数は、浮動小数点誤差によって期待値と完全には一致しない場合があります。
そのため、`===`で直接比較するのではなく、差が十分に小さいかを確認する方法があります。

```js
const result = 0.1 + 0.2;
const expected = 0.3;

console.log(Math.abs(result - expected) < Number.EPSILON);
```

このコードでは、2つの小数の差が非常に小さいかを確認するために`小数の比較`を行っています。

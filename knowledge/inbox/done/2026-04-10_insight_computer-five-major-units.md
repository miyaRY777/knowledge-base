{- [ ] `5大装置`とは？  👉 **コンピュータを動かすための基本となる5つの役割のこと**  **解説：** 5大装置とは、入力装置・出力装置・記憶装置・制御装置・演算装置のことです。 コンピュータは見た目が違っても、基本的にはこの5つの役割で動いています。 「どの部品が何を担当しているか」を整理するときの土台になります。  具体例：  ```ruby five_units = [   "入力装置",   "出力装置",   "記憶装置",   "制御装置",   "演算装置" ]  puts five_units ```  このコードでは、コンピュータの基本となる 5大装置 を並べて確認するために使っています。  ---  - [ ] `入力装置`とは？  👉 **コンピュータにデータや命令を入れるための装置**  **解説：** 入力装置は、人がコンピュータに情報を渡すための装置です。 キーボードやマウスなどが代表例です。 「コンピュータに何かを伝える側」と覚えると整理しやすいです。  具体例：  ```ruby input_devices = ["キーボード", "マウス", "マイク"] puts input_devices ```  このコードでは、代表的な 入力装置 を確認するために使っています。  ---  - [ ] `出力装置`とは？  👉 **コンピュータの処理結果を外に見せるための装置**  **解説：** 出力装置は、コンピュータが処理した結果を人が見たり聞いたりできる形にする装置です。 ディスプレイやスピーカー、プリンタなどがこれに当たります。 「コンピュータの答えを外に出す役割」です。  具体例：  ```ruby output_devices = ["ディスプレイ", "プリンタ", "スピーカー"] puts output_devices ```  このコードでは、代表的な 出力装置 を確認するために使っています。  ---  - [ ] `記憶装置`とは？  👉 **データやプログラムを覚えておくための装置**  **解説：** 記憶装置は、入力されたデータやプログラム、処理結果などを保存するための装置です。 大きく分けて、主記憶装置（メモリ）と補助記憶装置（ストレージ）があります。 一時的に使う場所か、長く保存する場所かで役割が分かれます。  具体例：  ```ruby storage_types = {   main_memory: "メモリ",   secondary_storage: "ストレージ" }  puts storage_types ```  このコードでは、記憶装置 の種類を整理するために使っています。  ---  - [ ] `主記憶装置`とは？  👉 **CPUがすぐ使うデータを一時的に置く場所**  **解説：** 主記憶装置は、一般的にメモリやRAMと呼ばれるものです。 高速ですが、電源を切ると内容が消えるのが特徴です。 作業中のデータを一時的に置く「作業机」のような存在です。  具体例：  ```ruby main_memory = "RAM" puts "#{main_memory} は一時的にデータを置く場所" ```  このコードでは、主記憶装置 が一時的な作業領域であることを確認するために使っています。  ---  - [ ] `補助記憶装置`とは？  👉 **データを長く保存しておくための装置**  **解説：** 補助記憶装置は、一般的にストレージと呼ばれ、HDDやSSDが代表例です。 主記憶装置より遅いですが、電源を切っても内容が残ります。 ファイルやアプリを保存しておく場所として使われます。  具体例：  ```ruby secondary_storage = ["SSD", "HDD"] puts secondary_storage ```  このコードでは、代表的な 補助記憶装置 を確認するために使っています。  ---  - [ ] `制御装置`とは？  👉 **他の装置に指示を出す司令塔の役割**  **解説：** 制御装置は、記憶装置から命令を読み出して解読し、他の装置に指示を出します。 どの順番で何をするかを管理する役割です。 実際に計算するというより、「全体を動かす」側の機能です。  具体例：  ```ruby instruction = "データを読み込む" puts "制御装置は「#{instruction}」などの指示を出す" ```  このコードでは、制御装置 が他の装置に指示を出す役割であることを表しています。  ---  - [ ] `演算装置`とは？  👉 **計算や比較などの処理を実行する装置**  **解説：** 演算装置は、足し算や引き算だけでなく、条件の比較なども行います。 制御装置からの指示に従って、実際の計算処理を担当します。 「考えて計算する部分」とイメージするとわかりやすいです。  ```ruby a = 10 b = 5  puts a + b puts a > b ```  このコードでは、演算装置 が行う算術演算や比較のイメージを表すために使っています。  ---  - [ ] `CPU`とは？  👉 **制御装置と演算装置をまとめた中心の装置**  **解説：** CPU は、制御装置と演算装置をひとつにまとめた中央処理装置です。 コンピュータ全体の処理の中心で、命令を読み取って実行します。 よく「コンピュータの頭脳」と説明されます。  具体例：  ```ruby cpu = {   control_unit: "命令を読み取って指示を出す",   alu: "計算や比較を行う" }  puts cpu ```  このコードでは、CPU が制御装置と演算装置で構成されることを整理するために使っています。  ---  - [ ] `フォン・ノイマン型アーキテクチャ`とは？  👉 **今の多くのコンピュータの土台になっている基本設計**  **解説：** フォン・ノイマン型アーキテクチャは、コンピュータを5つの役割で考える基本設計です。 現在の多くのコンピュータは、この考え方をもとに作られています。 5大装置を学ぶときの前提になる考え方です。  具体例：  ```ruby architecture = "フォン・ノイマン型アーキテクチャ" puts "#{architecture} をもとに5大装置を考える" ```  このコードでは、5大装置 の考え方の土台として フォン・ノイマン型アーキテクチャ を確認するために使っています。}

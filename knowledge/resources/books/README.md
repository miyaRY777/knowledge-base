# Books

`knowledge/resources/books/` は、書籍の読書メモを保管する場所。

## 置くもの

- 本ごとの読書メモ
- 章ごとの読書メモ
- Rails / Ruby などの用語メモ
- 後で見返したいポイント
- 自分の理解や疑問
- distill 候補

## ディレクトリ構成

本ごとにディレクトリを作り、必要に応じて章ごとにファイルを分ける。

```text
knowledge/resources/books/
└── book-title/
    ├── index.md
    ├── chapter-01.md
    ├── chapter-02.md
    └── chapter-03.md
```

## `index.md` の役割

`index.md` は、その本全体の入口になるファイル。

本の基本情報、読む目的、章メモへのリンク、全体メモ、distill 候補をまとめる。

## 用語メモを書くときの注意

- 文体はやさしく、簡潔にする
- 最初に「ひとことで意味」を書き、そのあとに「解説」を書く
- 解説は短めにする
- 用語名は必ずバッククォート付きで見出しに入れる
- コード例は JavaScript に固定しない
- Rails や ViewHelper なら `erb`
- Ruby の文法やメソッドなら `ruby`
- SQL なら `sql`
- JavaScript や Stimulus なら `js`
- 設計思想、概念、状態、役割、責務、アーキテクチャ、セキュリティ用語などは、無理にコード例を作らない
- コード例がその用語の理解に役立たない場合は、具体例を優先する
- 事実があいまいな場合は断定しない
- Rails の標準機能ではない名前については、「Railsの特別な機能名ではない」と明記する

## 運用ルール

- 本文をそのまま長くコピペしない
- 自分の言葉で要約する
- 本から得た学びを `knowledge/notes/` に直接保存しない
- atomic note にしたい学びは、後で `distill` して `knowledge/notes/` に昇格する
- 既存ノートと同じ概念なら、新規作成より既存ノート更新を検討する
- 似ているが観点が違う場合は、別ノートにして `Links` でつなぐ

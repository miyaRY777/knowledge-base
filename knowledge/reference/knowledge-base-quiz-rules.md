# knowledge-base クイズ運用ルール

## `/quiz yesterday`

- 前日に `capture` して `distill` した内容を復習対象にする
- 前日学習した各ノートから最低 1 問は出す

## `/quiz today`

- その日に `capture` して `distill` した内容をすべて復習対象にする
- 1 ノートにつき最低 1 問は出す
- ノート数が多い日は、各ノートから 1 問ずつ順番に出す
- その日の全ノートを一巡するまでクイズを続ける
- 最後に、その日の要復習ポイントを短くまとめる

## クイズ中の復習

- 間違えた問題や迷った問題は、その場で復習する
- 正解の確認、間違えやすい理由の整理、同じ論点の類題 1 問、の順で進める
- 正解できたら次の問題へ進む

## 間違い記録（quiz_fail_log）

- 間違えた問題の元ノートのフロントマターに、間違えた日付を `quiz_fail_log` へ追記する
- Phase 1（4択）・Phase 2（説明）・スキップのいずれでも記録する
- `quiz_fail_log` はリセットしない（累計記録）

```yaml
quiz_fail_log: [2026-06-10, 2026-06-18]
```

## 間違い記録を使った絞り込み

`/quiz` コマンドの引数で絞り込める:

| 引数 | 意味 |
|------|------|
| `fail=1` | ちょうど1回間違えた問題 |
| `fail>=2` | 2回以上間違えた問題 |
| `fail-in:2026-06` | 6月に間違えた問題 |
| `fail-in:2026-06=1` | 6月にちょうど1回間違えた問題 |
| `fail-in:2026-06>=3` | 6月に3回以上間違えた問題 |

## フェーズ管理

| フェーズ | 出題形式 | 昇格・降格条件 |
|---------|---------|--------------|
| Phase 1（4択） | 選択肢A〜D | 2連続正解 → Phase 2 へ昇格 |
| Phase 2（説明） | 自由記述 | 2連続不正解 → Phase 1 へ降格 |

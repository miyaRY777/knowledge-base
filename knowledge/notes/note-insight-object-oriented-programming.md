## Summary
- オブジェクト指向は、データと処理をひとまとまりのオブジェクトとして扱う考え方。
- Ruby ではクラスからインスタンスを作り、状態と振る舞いをまとめる。
- Rails でも Model などの役割分担を理解する土台になる。

## Body
オブジェクト指向は、データだけ、処理だけを別々に考えるのではなく、状態と振る舞いを持つ「もの」としてまとめて扱う考え方です。Ruby ではクラスを定義し、そこからインスタンスを作ることで、データと処理を同じまとまりとして扱えます。

Rails では、たとえば `User` や `Post` のような Model が、データの形や振る舞いを表す単位になります。何でも1か所に書くのではなく、役割ごとに責務を分けて考えると見通しがよくなります。

## Example
**具体例：**
- `User` は、名前やメールアドレスなどの状態を持つ
- `User` には、ユーザーに関係する処理をまとめられる
- `Post` は、タイトルや本文など投稿に関係する状態を持つ

この例では、データと処理を役割ごとにまとめるためにオブジェクト指向が関係しています。

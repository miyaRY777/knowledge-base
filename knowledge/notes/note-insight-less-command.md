## 背景

Linuxでは、ログファイルや設定ファイルなど何百行・何千行にもなるテキストファイルを頻繁に扱います。`cat` コマンドで一度に全表示すると画面が一気に流れてしまうため、**必要な部分だけ少しずつ表示しながら読む**ための仕組みが必要になりました。

## 結論

**`less` コマンドとは、大きなテキストファイルを1ページずつ読みやすく表示・閲覧するためのページャーコマンドです。**

## 理由

`cat` は全行を一気に表示するため、大きなファイルでは目的の場所を探すのが困難になります。`less` を使うと以下ができます。

- 上下にスクロールできる
- 文字列を検索できる
- 必要な部分だけ読める

## 具体例

```bash
less sample.txt
less log/development.log
```

### よく使う操作

| キー | 動作 |
|------|------|
| ↓ / ↑ | 1行下・上へ |
| Space | 次のページへ |
| b | 前のページへ |
| /文字 | 検索 |
| n | 次の検索結果 |
| q | 終了 |

## `cat` との使い分け

| コマンド | 向いている場面 |
|---------|--------------|
| cat | 短いファイル |
| less | 長いファイル（ログ・設定ファイルなど） |

## よくある勘違い

- **編集はできない** — 閲覧専用。編集には `nano` や `vim` を使う
- **cat の上位版ではない** — 目的が異なる。一気に表示したいなら `cat` の方が適切

## まとめ

`less` は大量のテキストを「本を読むように1ページずつ」閲覧できるコマンド。終了は `q`、検索は `/`、次の一致箇所へは `n`、次ページは `Space`、前ページは `b`。
